# Reviews

Code reviews, documentation reviews, and other analysis artifacts.

## File Naming

Format: `yyyy-mm-dd-<review-type>.md`

Examples:
- `2026-01-26-code-review.md`
- `2026-01-26-security-review.md`
- `2026-01-26-architecture-review.md`

## Index

| Date | Review Type | Status | Notes |
|------|-------------|--------|-------|

Status values:
- Open: Review findings not yet addressed
- Actioned: Review findings have been addressed
